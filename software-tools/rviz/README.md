# RViz

**RViz** (short for **ROS Visualization**) is the primary 3D visualization tool used in the Robot Operating System (ROS) ecosystem. Using various plug-ins, it's possible to visualize a robot's model, meshes, and the reference frames that make up the robot, facilitating the design and programming of multiple robots.

It also allows you to visualize information from different sensors such as cameras, lasers, LiDAR, Kinect devices, and others.

In simple words using RViz we can visualizate what the robot is "thinking" and "seeing" by pulling data from various sensors and software modules into a single 3D environment. Think of it as a **debugging window for the physical world**. While a simulator like Gazebo mimics physics and gravity, RViz simply displays the data that already exists in your ROS system.



Key Capabilities
RViz is highly modular; you can add "Displays" for different types of data. Common visualizations include:

* Point Clouds: Visual representations of 3D data from LiDAR or depth cameras (like an Xbox Kinect).
* Robot Models (URDF): A 3D model of the robot that moves its joints in real-time based on tf (transform) data.
* Laser Scans: 2D "slices" of the environment showing distances to obstacles.
* Maps: 2D or 3D grids generated by SLAM algorithms.
* Path Planning: Lines showing the planned route and arrows showing the robot's current goal (pose).

Why use it?
Without RViz, debugging a robot would involve staring at endless streams of text and numbers in a terminal. RViz lets you immediately spot errors, such as:

A sensor that is mounted upside down (the data will appear upside down in 3D).

A localization drift where the robot thinks it’s in a wall.

A path planner that is trying to drive through an obstacle.
